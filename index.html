Introduction

The auroras.live API allows you to obtain key aurora hunting information and access extensive data archives. It's also the system used by the auroras.live website, Pebble watch app and mobile apps

Usage

Data can be passed to the API via GET or POST, using the endpoints listed below

Endpoints

Non secure

http://api.auroras.live/v1/

Secure

https://api.auroras.live/v1/

About Timezones

By default, all dates and times returned are in UTC. By appending tz to your request, you can convert those times and dates into your local time. tz is an offset, in minutes. If you're accessing the API via Javascript, you can simply use new Date().getTimezoneOffset()

Example

http://api.auroras.live/v1/?type=ace&data=bz&tz=-660

Returns

{
  "date": "2016-03-31T02:53:16+11:00",
  "bz": "1.7",
  "colour": "yellow"
}

About Formatting (BETA)

By default, all data returned will be in JSON format. You can return data as XML by appending format=xml to your request

Please note that XML may not work on all types, for exampmle ace/threeday will return unexpected data. These issues are being investigated

Example

http://api.auroras.live/v1/?type=ace&data=bz&format=xml

Returns

<data>
    <date>2016-03-30T16:00:28+00:00</date>
    <bz>2.5</bz>
    <colour>yellow</colour>
</data>

About Colouring

Many data types will return a colour along with the information. This is to give hunters a relative idea of activity, and also to help programmers write code without having to add in lengthy checks to map values to colours. You can change the colour format returned by appending colour=hex or colour=rgb to the end of your request

Modules

There are a few "modules" that the API exposes that you can access, and they're listed below. Pay special attention to spelling, as words are spelled using British English (e.g. colour)

All Module

The "All" module bundles up a variety of information from the other modules and returns it to the user. This data is most commonly used for apps and web services, where you don't want to make multiple calls to retrieve data. The data returned includes

  • ACE data (speed, density, Bz, Kp, Kp 1 hour, Kp 4 hour)
  • Three day Kp forecast
  • Weather for the specified location
  • 9 day weather forecast
  • Probability of seeing an aurora overhead, along with calculated data showing possible auroral activity within a 9 degree (approximately 500km) radius, and the coordinates of the location with the highest chance of seeing an aurora overhead

Usage

http://api.auroras.live/v1/?type=all&lat=-37.663712&long=144.844788

Required Parameters

lat

The latitude of the area you wish to obtain information for

long

The longitude of the area you wish to obtain information for

Optional Parameters

ace

Must be true or false. Defaults to true. If false, data from SWPC is not included

weather

Must be true or false. If false, weather information (including forecast) is not included

ACE Module

The ACE module is the core of the API. It returns information obtained from ACE spacecraft, though it also returns information from the NOAA SWPC. This information includes the basic elements used in aurora hunting, such as Bz, speed, density and Kp

Usage

http://api.auroras.live/v1/?type=ace&data=kp

Parameters

data

Required. The type of information you wish to retrieve. Valid data types include:

  • bz - Bz, which is the direction of the interplanetary magnetic field (IMF)
  • density - Proton density
  • speed - Proton speed
  • kp - Current, 1 hour and 4 hour Kp readings
  • threeday - Three day Kp forecast
  • nowcast - Retrieves aurora visibility predictions
  • all - Everything above, except for nowcast

Details

bz

The direction of the Interplanetary Magnetic Field (IMF) in nanoteslas

Requires no other parameters

Example

http://api.auroras.live/v1/?type=ace&data=bz
{
  "date": "2016-03-31T07:05:37+00:00",
  "bz": "-1.5",
  "colour": "yellow"
}

density

Proton density in p/cc

Requires no other parameters

speed

Speed of the particles in km/s

Requires no other parameters

kp

Current Kp, as well as 1 hour and 4 hour Kp predictions

Requires no other parameters

nowcast

A percentage value indicating how likely you are to see an aurora directly overhead. Auroras.live does further calculations to determine how likely you are to see an aurora within 500km. This is useful for Australians who generally can only see auroras towards the horizon

Required Parameters

lat - Current latitude, -90 to 90
long - Current longitude, -180 to 180
